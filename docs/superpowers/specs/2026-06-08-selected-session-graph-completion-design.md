# Selected Session Graph Completion Design

## 背景

全局图谱已经可以显示多个会话中的学习节点、来源节点、随笔节点和闲聊便签节点。但历史会话里仍然可能存在未整理的学习内容：有些是正式学习过程，有些只是闲聊里冒出的学习线索，有些则能补充已有节点的子节点或关系。

这个功能不是“全局自动回填”，也不是静默扫描全部历史。它是一个由用户主动触发、可勾选会话、可预览结果的图谱补全工具。

## 目标

在全局图谱页新增入口：

```text
整理会话线索
```

点击后打开弹窗，让用户选择要扫描的历史会话。系统只扫描被勾选的会话，生成一批预览结果，用户确认后再写入图谱数据。

扫描结果可以不只生成闲聊便签，也可以生成或更新正式学习节点，并补充子节点和关系。

## 非目标

- 不自动扫描所有历史会话。
- 不在用户确认前写入节点。
- 不删除旧节点、旧证据或旧关系。
- 不发布、不改更新源、不改版本发布策略。
- 不实现完整手动节点编辑器。
- 不把纯闲聊、角色确认、生活隐私细节变成学习节点。

## 入口与流程

### 入口

入口放在全局图谱页，作为图谱工具按钮：

```text
整理会话线索
```

按钮只在全局图谱上下文出现。会话内图谱不默认显示这个入口，避免用户误以为它只处理当前会话。

### 弹窗第一步：勾选会话

弹窗列出历史会话，每个会话显示：

- 会话标题
- 最近更新时间
- 消息数量
- 可能学习轮次数量
- 现有图谱节点数量
- 复选框

底部操作：

```text
预览线索
```

默认不全选。可以提供“全选当前筛选结果”，但仍然需要用户确认后才扫描。

### 弹窗第二步：预览结果

扫描完成后不直接落库，而是进入预览页。预览按结果类型分组：

- 将新增的学习节点
- 将更新的已有学习节点
- 将补充的子节点 / 关系
- 将新增的闲聊便签
- 疑似重复，准备合并

每条预览显示：

- 节点标题
- 节点类型
- 来源会话
- 原话或证据片段
- 可能的大纲路径
- 合并目标，如果命中已有节点
- 本次会补充什么

底部操作：

```text
保存到图谱
```

## 产物类型

### 1. 正式学习节点

当历史会话中出现真实学习行为时，可以生成或更新正式学习节点。

真实学习行为包括：

- 解释概念
- 做题或练习
- 纠错
- 复述
- 追问
- 举例
- 迁移应用
- 暴露卡点

正式学习节点可以进入掌握度、证据、卡点、下一步建议等 L1 学习节点体系。

### 2. 闲聊便签节点

当历史会话只是提到学习线索，但没有进入学习过程时，生成闲聊便签节点。

示例：

```text
我最近解析几何里的椭圆总是搞混。
```

如果后续没有解释、练习、纠错或追问，这不应该直接成为正式学习节点，而应成为闲聊便签。

### 3. 子节点与关系补充

如果已有父节点，但历史会话中出现了未生成的子节点或关系，系统应补充它们。

示例：

```text
已有：解析几何
扫描发现：椭圆、双曲线、抛物线、离心率、焦点、标准方程
```

处理结果可以是：

```text
解析几何
  -> 圆锥曲线
    -> 椭圆
    -> 双曲线
    -> 抛物线
  -> 坐标法
```

但每个新增项的节点类型仍要根据证据强度判断：

- 已讲解或练习过：正式学习节点
- 只是提到：闲聊便签或候选节点
- 来源结构里出现：结构 / 候选节点

## 去重与合并规则

### 核心原则

只能合并，不做删除式替换。

重复节点命中后，不新建重复节点。保留原节点身份、历史证据、会话来源和已有关系。

### 匹配依据

判断是否为重复或同一主题簇时，综合使用：

- 规范化标题
- 大纲路径
- 节点类型
- 关键词
- 相关父节点
- 来源会话中的语义证据

示例：

```text
椭圆
解析几何 / 圆锥曲线 / 椭圆
椭圆标准方程
```

这些可能属于同一主题簇，但不一定全部是同一个节点。系统应优先合并明显重复项，保留有必要区分的子节点。

### 当前版本的覆盖规则

当前产品还没有手动编辑节点功能，所以所有节点内容都视为系统自动生成内容。

在命中重复节点时，以下自动生成字段可以被新扫描结果覆盖：

- 标题
- 摘要
- 大纲路径
- 节点类型判断
- 主要卡点
- 下一步建议
- 相关关键词
- 结构化关系标签

以下内容不覆盖，只追加并去重：

- 证据片段
- 来源会话
- 练习 / 追问记录
- 历史关系

关系使用 upsert：

- 已有关系：更新权重或关系类型
- 新关系：补充写入
- 本次没扫到的旧关系：保留，不删除

### 未来手动编辑后的规则

后续如果新增手动编辑节点功能，需要给节点增加保护标记，例如：

```text
user_edited_at
user_locked_fields
```

只要用户手动改过节点内容，后续扫描命中该节点时不能静默覆盖用户修改。需要弹窗提醒：

```text
系统发现历史会话里有一版更完整的节点内容。

你之前手动改过这个节点：
[保留我的修改]
[合并系统建议]
[查看差异]
```

当前版本不实现这层保护，因为还没有手动编辑入口。

## 写入策略

用户点击“保存到图谱”后，系统按预览结果写入。

写入动作包括：

- 新增正式学习节点
- 更新已有正式学习节点
- 新增闲聊便签节点
- 更新已有闲聊便签节点
- 补充子节点
- upsert 关系
- 追加证据和来源

写入必须是幂等的。同一批会话重复扫描，不应产生重复节点或重复证据。

## 边界过滤

以下内容不应生成节点：

- 纯角色确认，例如“你是谁”“我是你的学生吗”
- 纯寒暄
- 纯情绪表达且没有学习对象
- 生活隐私细节
- 没有学习含义的系统过程词
- 和当前学习目标无关的闲聊

标准隐私级别允许记录学习偏好、卡点、掌握进度和学习启发。严格隐私级别下，只允许记录明确学习内容，不记录生活细节。

## 成功标准

- 用户可以在全局图谱页点击“整理会话线索”。
- 用户可以勾选指定历史会话，而不是被动扫描全局。
- 扫描结果先预览，确认后才写入。
- 结果可以生成正式学习节点，也可以生成闲聊便签。
- 重复节点默认合并，不新建重复项。
- 当前版本可以覆盖系统自动生成字段。
- 证据和关系追加去重，不删除旧内容。
- 已有父节点缺子节点时，可以补充子节点和关系。
- 没有学习含义的闲聊不会被生成节点。

## 验证要求

最低验证：

- 选定一个会话扫描后，能预览候选结果。
- 用户取消弹窗时，不写入任何节点。
- 用户确认后，写入学习节点和闲聊便签。
- 重复扫描同一会话，不产生重复节点。
- 已有节点命中时执行合并覆盖自动字段，而不是新建。
- 证据追加去重。
- 旧关系未被删除。
- 全局图谱能显示新增节点和补充关系。
