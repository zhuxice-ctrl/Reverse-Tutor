# 闲聊便签节点设计

## 背景

手机端图谱不能把闲聊里的所有内容都当成已学习知识点，也不能完全丢掉闲聊中自然冒出来的学习线索。

标准隐私级别的含义是：正常记录学习偏好、错因、掌握进度和有学习价值的闲聊启发；不扩展生活隐私细节，不把纯角色玩笑、身份确认或寒暄变成知识节点。

## 目标

新增一种 L2 节点：闲聊便签节点。

它是一张清晰的学习便签，用来记录闲聊中出现的学习线索，并允许用户把这条便签归入学习大纲。它不是状态面板，不是诊断报告，也不是已学习知识点。

## 节点定义

闲聊便签节点来自轻聊天或自然对话中的学习线索，例如：

- 学习主题：`て形`、`动词变形`、`N2 听力快语速`
- 潜在卡点：`聞こえる / 聞かれる 分不清`
- 方法线索：`shadowing`、`慢速跟读`
- 可迁移例子：`用买便当的例子讲语法`

以下内容不应生成闲聊便签节点：

- `师生角色定位`
- `你是谁我是谁`
- 纯寒暄、纯情绪、纯角色玩笑
- 生活隐私细节
- 没有明确学习含义的系统过程词

## 与学习节点的区别

闲聊便签节点不显示掌握度，不显示 `0%`，不显示“证据不足”，不展示平均正确率或平均深度。

它只回答一个问题：

> 这条闲聊里冒出来的线索，为什么值得记下来？

正式学习节点回答的是：

> 你现在学得怎么样，卡在哪里，下一步怎么突破？

两者不能混用。

## 图谱表现

闲聊便签节点在图谱上应比学习节点轻：

- 类型标签：`闲聊便签`
- 视觉感觉：便签、线索、提醒，而不是诊断卡
- 可与当前会话、大纲分类、相关学习节点连接
- 不参与 mastery 统计
- 不影响平均掌握度

示例关系：

```text
当前会话
  -> 闲聊便签：て形总是写乱
  -> 闲聊便签：shadowing
  -> 闲聊便签：聞こえる / 聞かれる 分不清
```

如果存在大纲分类：

```text
闲聊便签：て形总是写乱
  -> 归入：日语 N2 / 语法 / 动词变形
```

如果后续正式学习命中：

```text
闲聊便签：て形总是写乱
  -> 来源线索
正式学习节点：て形
```

## Bottom Sheet 设计

弹窗应像一张便签，避免器械式字段。

示例：

```text
て形总是写乱

闲聊里冒出来的一个小卡点。
你提到「する」「くる」和五段动词混在一起时容易写错，
这更像是一个该收进「日语 N2 / 语法 / 动词变形」的小线索。

[归入大纲]   [围绕它聊]
查看原话
```

不显示：

- `来自：半年通过 JLPT N2 · 第 8 轮闲聊`
- `待接入学习`
- 掌握度
- 当前判断
- 平均正确率
- `来一题`

“查看原话”可以存在，但它是一个轻动作，不是主信息。

## 详情页设计

详情页仍然是便签，不是报告。

结构：

```text
て形总是写乱

为什么记下它
你在闲聊里提到，动词变形混在一起时容易写错。
这可能不是单纯聊天内容，而是一个后续值得处理的小卡点。

可能放到哪里
日语 N2 / 语法 / 动词变形

原话
“する、くる 和五段动词混在一起时，我总下意识写错。”

可以怎么用
- 归入大纲，作为后续学习线索
- 围绕它继续问
- 等正式学习时转成学习节点

相关便签
shadowing
聞こえる / 聞かれる
N2 听力快语速
```

详情页不展示学习节点的七模块诊断，不展示证据不足，不展示 mastery。

## 归入大纲

`归入大纲` 的含义是：

> 把这条闲聊便签作为锚点，归类到学习大纲里的某个位置。

它不是：

- 立即创建正式学习节点
- 立即进入诊断
- 立即计算掌握度
- 立即安排练习

归入后，便签应保存目标分类，例如：

```text
闲聊便签：て形总是写乱
归入：日语 N2 / 语法 / 动词变形
```

后续用户围绕它学习时，系统可以用这个归类帮助建立学习节点和关系。

## 升级规则

闲聊便签可以升级为正式学习节点，但必须有学习行为触发。

触发条件：

- 用户明确围绕便签提问
- 后续 `study_full` 回合命中它
- 出现解释、练习、纠错、复述、迁移等学习证据

升级后：

- 正式学习节点获得掌握度、卡点和证据
- 原便签保留为来源线索
- 图谱中建立从便签到学习节点的关系

## 生成规则

轻聊天回合可以提取闲聊便签，但应遵守：

- 只提取有明确学习含义的词或短语
- 优先提取 1 到 4 个高质量线索
- 不提取角色元话题
- 不提取隐私生活细节
- 不写入 mastery
- 不改变现有学习节点规则

示例输入：

```text
最近卡在「て形」变形上，尤其是「する」「くる」和五段动词混在一起时容易写错。
听力一到早口の会話就听不出来。
```

应生成：

```text
闲聊便签：て形总是写乱
闲聊便签：N2 听力快语速
```

不应生成：

```text
已学习知识点：て形 · 0%
当前判断：证据不足
师生角色定位
```

## 成功标准

- 闲聊里的学习启发能进入图谱。
- 这些节点不再显示 mastery、0%、证据不足或诊断报告。
- `师生角色定位` 等角色元话题不会成为节点。
- `归入大纲` 明确表示把便签作为锚点归类到学习大纲。
- 全局图谱能显示多个会话的闲聊便签和学习节点。
- 会话内点击闲聊便签时，弹窗像便签，而不是状态面板。

## 非目标

- 不做完整大纲编辑器。
- 不自动发布或改更新源。
- 不删除原有学习节点、资料节点、随笔节点系统。
- 不把轻聊天统一改成完整学习链。
