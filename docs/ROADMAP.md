## 0. 现状基线（2026-05）

- **V1 学习闭环**：评估→决策→行动、掌握度/复习调度、追问/考官/线索/脚手架学生角色——已上线（客户端 v0.18.0 APK）。

## 1. 北极星与原则

**北极星**：让用户**通过教 AI 真正学会**，并逐步从"被动陪练"演进为"主动感知 + 多模态 + 长期陪伴"的个人 AI——**始终本地优先、隐私边界清晰**。

**贯穿原则**（来自 `01-新架构总览`，不可漂移）：
1. 三模式（学习 / 目标 / 陪伴）**共用统一编排器**，差异只在 Mode Policy。
2. **安全与权限门永远在最前**，尤其陪伴模式与屏幕识别。
3. **长期记忆分层**：episode 可审计 + 时间演化图谱（关系演化）+ 学习状态表（掌握度/复习）。
4. **可审计过程**：每轮决策有"本轮判断"，用户可见、可追证据。
5. **双实现对齐**：能力进 APK 必须改客户端 JS；后端是规格与回归基线。

**优先级（已与用户确认）**：**教学深度 → 多模态 → 陪伴/主动观察**。
理由：当前实机测的是学习模式、教学有效性是产品命脉；主动观察隐私面最大，最该压后做扎实。

## 2. V3 — 教学深度版（Teaching Depth）

**主题**：把"教即是学"做到**真正有效且可信**——让用户被引导着自己发现错误、补齐前置、形成可迁移的理解。

**北极星指标**
- 用户讲错时被引导改正的比例（纠错命中率）。
- 掌握度"真实性"：标记掌握的知识点在延迟复习/迁移题上的保持率。

**关键能力线**
1. **反例追问式纠错**（进行中）：`evaluation.misconception` + `challenge` 动作 + `confused_student` 角色 + 升级阶梯 + 可调档 `correction_persistence`。
2. **图谱驱动的前置补缺**：检测当前知识点缺失的前置（KG `前置于` 边），在学生口吻下自然回补。
3. **自适应难度与追问深度**：依据掌握度/情绪动态调 `difficulty`、`probing_intensity`，避免过载或太浅。
4. **掌握度证据模型升级**：区分 explanation / retrieval / transfer / delayed_retrieval / correction 的权重，弱化"嘴上说懂"。
5. **复习调度打磨**：到期复习 1/3/7/14 阶梯与图谱挂起复习联动，主动但不打扰。

**验收信号**：实机连续教错一个概念，AI 能在保持学生人设下用反例把用户带到自我修正；掌握度不再被"我懂了"轻易拉高。

**范围边界（V3 不做）**：不做图片/语音输入，不做屏幕观察，不做陪伴情绪机。

## 3. V4 — 多模态学习版（Multimodal Learning）

**主题**：让用户能**拍照/截图把现实题目和教材喂进来**，把 `01` 流程图里"题目生成/批改、屏幕识别"从埋点变成实际工具。

**北极星指标**
- 拍照识题→可对话/可纠错的成功率。
- 题目批改的有用性（用户采纳率）。

**关键能力线**
1. **拍照识题 / 教材页识别**：图片→题干/知识点（已有 `vision.py`、`ImageExtract` 基础），结构化进会话与图谱。
2. **题目生成与批改**：围绕当前知识点出题、批改、定位错因，回写 error_log / 图谱。
3. **手写与公式**：数学公式、手写步骤识别（评估可行性，可能依赖外部模型）。
4. **图片→图谱**：识别内容沉淀为概念/错因节点，参与长期复习。
5. **（评估）语音**：仅当能做到本地化/低延迟才投入，否则不作为重心。

**验收信号**：拍一道数学题，AI 能识别题干、以学生身份请教并在你讲错时纠错；生成同类题让你练。

**范围边界（V4 不做）**：不主动截屏/观察（那是 V5）；语音不作为硬性目标。

**依赖/风险**：多模态模型的本地可用性与成本；离线客户端如何处理（可能需要"配置了视觉模型才启用"的降级路径）。

## 4. V5 — 陪伴与主动观察版（Companion & Proactive Observation）

**主题**：`24` 文档原 V3 占位——从"用户主动问"走向"AI 主动感知并适时介入"，同时深化陪伴模式。**隐私面最大，做之前先把权限门做厚。**

**北极星指标**
- 主动介入的"有用且不打扰"评分。
- 陪伴模式的情绪连续性 / 长期记忆边界的安全性（零越界事故）。

**关键能力线**
1. **权限与隐私门加厚**（前置必做）：屏幕/窗口观察黑名单、手动识别开关、可审计的观察日志、随时一键关停。
2. **屏幕/窗口观察**：在白名单与用户显式授权下，识别当前界面并提供学习/任务帮助。
3. **相册/图片手动识别流程**：用户主动选图→识别→入记忆，全程手动可控。
4. **陪伴模式长期记忆边界**：情绪/陪伴记忆的保留与遗忘规则、与学习记忆隔离。
5. **主动关怀/提醒**：定时复习、目标推进、情绪关怀的主动触发，遵守 proactivity 与隐私级别。

**验收信号**：在用户显式授权下，AI 能观察到"我在看某页 PDF"并主动以学生身份发问；用户可随时看到/关停观察，无任何越界记录。

**范围边界**：默认全部关闭、opt-in；不做后台静默上传；不碰黑名单内窗口。

**依赖/风险**：平台权限（Android/桌面）、隐私合规、用户信任——**这是最需要谨慎和分阶段灰度的版本**。

## 5. V6 — 商业化与分发（Horizon）

> 用户已确认有商业化倾向，目标形态：**微信小程序**。可推迟到产品成熟后做，但**不是"换壳重打包"**，需提前留对冲。

**目标**：在本地优先的内核之上，扩出可分发、可商业化的形态（首选微信小程序）。

### 5.1 微信小程序：现实约束（必须先认清）
- 小程序**不能直接跑现有 PWA**（自有 WXML/WXSS + 沙箱 JS，无 DOM、canvas 受限）→ 当前单文件 `index.html` 等于**前端重写**。
- 小程序**禁止前端直连第三方 LLM**（只能请求已备案的合法域名）→ **必须服务端代理**。
- 这恰好让现有 **Python 后端（已建、320 测试绿）成为生产服务端**，不浪费。
- 硬性合规门槛：**主体资质、ICP 备案域名、HTTPS、内容安全审核、AI 类目资质**。

### 5.2 现在就该做的低成本对冲（避免后期重写级阵痛）
1. **业务逻辑与渲染分层**：当前引擎逻辑与 DOM 缠在 `index.html` 里是最大隐患；新功能尽量把"引擎/状态"与"界面渲染"解耦。
2. **尽早定跨端形态**：要么采用 **Taro / uni-app**（一套代码出 小程序 + H5 + App），要么小程序前端重写 + 复用 Python 后端——此决策影响"现在怎么写客户端"。

### 5.3 其他远期候选（待拍板）
多端同步（本地优先 + 可选端到端加密）、可分享的知识图谱/课程模板、内容生态。**默认不做，除非明确要。**

> 结论：**小程序作为 V6 推迟可行**，但"只调打包"低估了它——把上面两条对冲提前做，后期迁移才便宜。

## 6. 跨版本工程基建线（每个大版本都要照顾）

1. **双实现对齐治理**：后端与 `index.html` JS 引擎的 prompt/动作/评估 schema 容易漂移——需建立"改一边必同步另一边"的清单（已写进 `AGENTS.md`），长期考虑抽取共享 schema/契约测试。
2. **测试纪律**：`py -m pytest`、mock LLM、回归基线保持绿；新能力先写测试。
3. **APK 发布管线**：固定签名、版本号三处同步（`mobile/package.json` / `build.gradle` / `CHANGELOG`）、实机验证清单。
4. **隐私门优先**：任何"主动/多模态/观察"能力进来前，先过安全与权限门设计评审。
5. **多 agent 协作**：Cascade 出 spec + 审查 + 回归，Codex 辅助实现；注意 Codex 残留进程会回滚编辑（见 `AGENTS.md`）。

## 7. 风险登记（Top）

| 风险 | 影响 | 应对 |
|---|---|---|
| 双实现漂移 | 后端测试绿但 APK 行为不一致 | 同步清单 + 长期契约测试 |
| 多模态模型本地可用性/成本 | V4 落地受限 | 设"配置视觉模型才启用"的降级路径 |
| 屏幕观察隐私/信任 | V5 失败甚至声誉风险 | 权限门前置、默认关闭、opt-in、可审计、灰度 |
| 教学有效性难量化 | 北极星无法验证 | 用延迟复习/迁移题做掌握度真实性度量 |
